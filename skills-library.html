Skills Library

Claude Code slash commands for the UX Library workflow. Install by dropping .claude/ contents into your project root. Skills with references or scripts are bundled as a zip — unzip directly into your project.

Figma Skills
/figma-use
command + 11 refs + working-with-design-systems/
Mandatory prerequisite — load before every use_figma call. Covers Plugin API rules: return pattern, page reset, color range (0–1), font loading, fill/stroke mutation, error recovery, and the full pre-flight checklist.
Figma .zip
/figma-generate-library
command + 7 refs + 9 scripts
Design system builder — orchestrates 20–100+ use_figma calls across 4 phases (Discovery → Foundations → File Structure → Components). Includes phase workflow, token architecture, naming conventions, and reusable helper scripts.
Figma .zip
/figma-create-design-system-rules
command + rule-template.md
Generates project-specific design system rules for CLAUDE.md, AGENTS.md, or Cursor .mdc. Analyzes the codebase and produces rules encoding component location, styling approach, token usage, and Figma-to-code workflow.
Figma .zip
Design System
/mui-design
command only
MUI design system foundations — full component catalog (39 components across 6 categories), theme token system (palette, typography, spacing, shape, breakpoints, z-index, transitions, shadows), Figma-to-MUI token mapping, component design rules, and spec-writing conventions for the Global section.
MUI .md
Documentation
/document-feature-spec
command only
Generates a complete feature.md + feature.html spec from a Figma frame URL. Reads product config files, maps tokens and copy, documents all states and components, and updates the coverage table, changelog, and sitemap.
Docs .md
/add-platform-delta
command only
Documents a platform-specific delta on top of an existing feature spec. Produces mobile-web.md, mobile-app.md, admin-portal.md, etc. with only the differences from the base feature.
Docs .md
/audit-figma
command only
Audits a Figma file against the UX Library specs. Compares copy, states, components, and platform rules. Outputs a prioritised findings report (Critical / Moderate / Minor) with actionable fix instructions.
Docs .md
/cross-product-check
command only
Compares a UX pattern across two or more products. Identifies global promotion candidates, per-product differences, conflicts, and documentation gaps. Token-efficient — reads index files and targeted sections only.
Docs .md
Workflow
/setup-product
command only
Scaffolds a new product section in the UX Library — creates the _index, config/, and changelog files, updates the nav, and adds the product to the sitemap.
Util .md
/promote-to-global
command only
Promotes a component or pattern from a product spec to /global/. Validates that the pattern is used in 3+ products, creates the global page, and updates all affected product specs to reference it.
Util .md
/run-skill-tests
command only
Runs a self-test suite against the active skills to verify they are functioning correctly. Checks that required files exist, references resolve, and the skill descriptions match their behavior.
