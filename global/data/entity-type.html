Entity Type

Shared entity type definitions used across all Colorkrew products. Defines what each entity is and how permission precedence works when scopes overlap.

Entity Types

Individual User
Entity
A single person with a Colorkrew account. Belongs to one Department and zero or more User Groups.
Group
Entity — two types
A collection of users. Exists as either a Department or a User Group.
Everyone
Scope
Company-wide scope. Applies to all users regardless of department or group membership.
Lowest precedence

Group Types

Type Description Membership per User
Department Organizational unit — e.g., Engineering, Sales, HR. Reflects the company org structure. Exactly one
User Group Flexible grouping for access control, workflows, or cross-functional collaboration. Zero or more

Permission Precedence

When permission scopes conflict, the most specific (smallest) scope always wins. An individual user's permission overrides any group or company-wide setting.

Example

Scenario — three overlapping permissions on the same resource
Everyone Read-only access
User Group Edit access
Individual User No access ← wins

The user's individual setting (no access) takes precedence over both the group and company-wide settings — even though a broader scope grants more access.

Rules

  • A user belongs to exactly one Department at a time — departments are mutually exclusive.
  • A user can belong to zero or more User Groups — there is no upper limit.
  • Individual User permissions always override Group permissions.
  • Group permissions always override Everyone permissions.
  • When a user is in multiple User Groups with conflicting permissions, the most restrictive applies — unless the product defines a different resolution strategy.
