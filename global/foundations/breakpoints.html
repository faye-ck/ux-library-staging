Breakpoints

MUI uses five breakpoints for responsive layouts. Always document designs mobile-first — specify behaviour from a minimum width, not below a maximum. Use theme.breakpoints tokens, never hardcoded pixel values.

Breakpoint Scale

Key Min width Typical target
xs 0px Small phones
sm 600px Large phones, small tablets
md 900px Tablets, small laptops
lg 1200px Desktops, wide laptops
xl 1536px Wide monitors

Mobile-first rule

Document breakpoints as "from sm up" (min-width), not "below sm" (max-width). MUI's breakpoint helpers all use min-width: theme.breakpoints.up('sm'), theme.breakpoints.down('md'), theme.breakpoints.between('sm','lg').

Common Layout Patterns

Breakpoint Nav Shell Content Grid columns
xs–sm Temporary drawer (hamburger) Full-width 4
md Persistent mini rail Offset by rail width 8
lg–xl Persistent expanded drawer Offset by drawer width 12

Rules

  • Never document a layout using only a desktop view — always note mobile behaviour
  • Drawer behaviour changes at md (900px) — see Nav Shell component
  • Typography, spacing, and touch targets should scale across breakpoints
