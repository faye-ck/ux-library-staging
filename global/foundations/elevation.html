Elevation

Controls perceived depth and layering. MUI uses a 25-step shadow scale (shadows[0]–shadows[24]) and named z-index layers. Never assign arbitrary box-shadow CSS or z-index values — always reference the scale.

Shadow Scale

Level Token Common use
0 shadows[0] No elevation — flat surfaces, disabled states
1 shadows[1] Cards at rest, text fields
2 shadows[2] Raised cards, contained buttons
4 shadows[4] App bar, drawer
8 shadows[8] Menus, popovers, tooltips
16 shadows[16] Modals, dialogs
24 shadows[24] Maximum — full-screen overlays

Z-Index Layers

Layer Value Component
mobileStepper 1000 MobileStepper
speedDial 1050 SpeedDial
appBar 1100 AppBar
drawer 1200 Drawer
modal 1300 Dialog, Modal
snackbar 1400 Snackbar
tooltip 1500 Tooltip

Rules

  • Use elevation to convey interaction depth — interactive surfaces lift on hover
  • Modals sit above drawers; drawers sit above the app bar
  • Never assign a custom z-index — reference the layer token
  • Product themes may reduce shadow intensity for flat/minimal styles — document in _theme.md
