Iconography

Tabler Icons is the primary icon library for all Colorkrew products. Icon style and stroke width are a product-level decision, but must be consistent within a product. Custom icons are allowed for Colorkrew-specific concepts that Tabler cannot represent.

Primary Library — Tabler Icons

Tabler Icons provides 5000+ icons as SVG, web font, and React components. It's free, open-source, and covers most standard UI needs.

Style Rules

Each product chooses one style and applies it consistently across all icons. Do not mix styles within a product.

Outline style
Lighter and more neutral. Good for dense UIs with many icons.
Filled style
More visual weight. Good for active states, emphasis, or consumer-facing products.
Rule Detail
Style Outline or filled — one style per product, chosen at the product level
Stroke width 1, 1.5, 2, or 2.5 — consistent within the product. Default is 2.
Color Inherit from surrounding text color, or use a MUI palette token. Never hardcode hex values.
Size Use standard sizes only (see below)

Custom Icons

Custom icons are permitted when a concept is unique to Colorkrew branding or product terminology and no suitable Tabler icon exists.

Custom icons must:
  • Match the product's chosen style (outline or filled) and stroke width
  • Use a clean 24×24px SVG viewBox
  • Be documented in the product's _theme.md or component spec with icon name and intended usage
  • Be reviewed and approved before use in production

Examples of valid custom icon needs: Colorkrew logo mark, product-specific concepts like "Biz Pass", unique workflow terminology that Tabler cannot represent adequately.

Accessibility

  • Decorative icons (next to visible label text): add aria-hidden="true".
  • Meaningful icons (no adjacent label): add aria-label to the icon or use a visually hidden <span>.
  • Interactive icon-only buttons: must include a visible or accessible label — tooltip alone is not sufficient.
  • Minimum touch target for icon buttons: 44×44px (see Accessibility).

Do / Don't

Do
  • Choose one style (outline or filled) per product and apply consistently
  • Use standard sizes: 16, 20, 24, 32, 48px
  • Match custom icon style and stroke width to the product
  • Add aria-hidden="true" to decorative icons
  • Document custom icons in _theme.md
Don't
  • Mix outline and filled icons within a product
  • Use icons smaller than 16px
  • Use inconsistent stroke widths within the same product
  • Use icons from other libraries without approval
  • Render icon-only buttons without an accessible label
