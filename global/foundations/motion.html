Motion

Motion should be purposeful and subtle. Use MUI transition tokens for consistency. All animations must respect prefers-reduced-motion — never use motion as the sole indicator of a state change.

Principles

01
Purposeful
Every animation communicates something — state change, hierarchy, or context shift. No purely decorative motion.
02
Subtle
Short durations and gentle easing. The UI should feel responsive, not theatrical or sluggish.
03
Accessible
Always apply prefers-reduced-motion media query. Users who disable animations must still get full functionality.
04
Consistent
Use the same duration and easing token for the same type of action across all components in a product.

Duration Tokens

theme.transitions.duration.<name>

Token Value Use
shortest150msMicro-interactions — hover states, focus rings
shorter200msSmall UI state changes — toggle, checkbox, chip
short250msDefault component transitions
standard300msPage-level transitions, modals, drawers, accordions
complex375msMulti-step or elaborate transitions, skeleton shimmer
enteringScreen225msElements entering the viewport
leavingScreen195msElements leaving the viewport (slightly faster than entering)

Easing Tokens

theme.transitions.easing.<name>

Token Curve Use
easeInOutcubic-bezier(0.4, 0, 0.2, 1)Default — most transitions in the middle of a flow
easeOutcubic-bezier(0.0, 0, 0.2, 1)Elements entering the screen (starts fast, settles gently)
easeIncubic-bezier(0.4, 0, 1, 1)Elements leaving the screen (starts slow, exits quickly)
sharpcubic-bezier(0.4, 0, 0.6, 1)Elements that may return — e.g., a collapsible drawer

Accessibility

Required: Wrap all transitions and animations in a @media (prefers-reduced-motion: reduce) block and set transition: none or animation: none. Some users experience nausea or seizures from motion — this is a WCAG 2.1 AA requirement.
  • Never rely on animation alone to communicate state — always pair with a color, text, or icon change.
  • Avoid content that flashes more than 3 times per second (WCAG 2.3.1).
  • See Accessibility for full guidelines.

Do / Don't

Do
  • Use MUI duration and easing tokens
  • Apply prefers-reduced-motion to all animations
  • Use transform and opacity for smooth animations
  • Pair motion with a visible state change (color, icon, text)
  • Keep transitions under 400ms
Don't
  • Use durations above 400ms for standard interactions
  • Animate layout properties like width, height, top, left
  • Add motion to non-interactive, informational elements
  • Use animation as the only indicator of a state change
