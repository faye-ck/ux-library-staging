Spacing

The base unit is 8px. All spacing — padding, margin, gap — uses multiples of 8. Fine-grained adjustments within components may use multiples of 4. Always reference theme.spacing() tokens in code.

Scale Overview

Two scales are supported. Use the primary scale by default.

Primary Scale ×8

Use for layout, section padding, card padding, page margins, and gaps between components.

8, 16, 24, 32, 40, 48, 64, 80, 96, 128

Allowed Scale ×4

Use for fine-grained adjustments within components — icon padding, dense lists, inline gaps, input fields.

4, 8, 12, 16, 20, 24, 28, 32

MUI Token Reference

theme.spacing(n) — where n × 8px = result. Use in sx props or theme.spacing() calls.

MUI Grid

The default MUI Grid spacing is spacing={2} = 16px gutters. Adjust with the same multiplier system. The MUI Grid uses a 12-column base by default.

When to Use ×4

  • Padding inside an icon button (e.g., 4px or 12px)
  • Gaps between a label and an icon in the same element
  • Dense list item padding where ×8 would feel too loose
  • Fine alignment corrections within a tightly constrained layout

Avoid mixing ×8 and ×4 at the same layout level. Use one consistently per component.

Do / Don't

Do
  • Use theme.spacing() in all code
  • Default to ×8 for layout and component gaps
  • Use ×4 within components for fine adjustments
  • Keep one scale consistent at each layout level
Don't
  • Hardcode px values (e.g., padding: 10px)
  • Use arbitrary values not on either scale (7px, 13px, 22px)
  • Mix ×4 and ×8 on adjacent elements at the same level
