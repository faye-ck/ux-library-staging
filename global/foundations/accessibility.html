Accessibility

Accessibility is a design requirement, not a post-launch fix. All Colorkrew products must meet WCAG 2.1 AA compliance. Build for keyboard navigation, screen readers, and users with motion sensitivity from the start.

Target: WCAG 2.1 Level AA — minimum standard for all products

Color Contrast

All text and meaningful UI elements must meet the following minimum contrast ratios against their background.

Element Minimum Ratio
Normal text (below 18px regular or 14px bold) 4.5:1
Large text (18px+ regular or 14px+ bold) 3:1
UI components — input borders, icons, focus rings 3:1
Disabled elements No requirement — but avoid making disabled states look interactive
  • Always verify contrast between palette.text.primary and palette.background.default.
  • Check palette.text.secondary on palette.background.paper too — this is commonly missed.
  • Never convey information by color alone — pair with a text label, icon, or pattern. See Color System.

Keyboard Navigation

Every interactive element must be reachable and operable without a mouse.

Key Expected Behavior
TabMove forward through interactive elements in reading order (left-to-right, top-to-bottom)
Shift+TabMove backward through interactive elements
Enter / SpaceActivate buttons, links, checkboxes, and form controls
EscapeClose modals, dialogs, drawers, and popovers
Arrow keysNavigate within radio groups, menus, tabs, sliders, and date pickers

Focus Management

  • Focus indicators must always be visible — minimum 3:1 contrast ratio against adjacent colors.
  • Never remove the default browser focus outline without providing a custom visible alternative.
  • On modal or dialog open: move focus to the first interactive element inside the overlay.
  • On modal or dialog close: return focus to the element that triggered it.
  • Trap focus within open modals and drawers so keyboard users cannot accidentally reach content behind the overlay.

Screen Reader Support

  • Use semantic HTML elements first — <button>, <nav>, <main>, <header> — before reaching for ARIA.
  • All images require alt text. Decorative images use alt="".
  • Every form input must have a visible or accessible label via <label>, aria-label, or aria-labelledby.
  • Icon-only buttons must include aria-label. A tooltip alone is not sufficient.
  • Decorative icons must have aria-hidden="true". See Iconography.
  • Use aria-live regions for dynamic content that updates without a page reload (e.g., toast notifications, inline validation).

Motion & Animation

  • Respect prefers-reduced-motion: reduce — disable or significantly reduce all transitions and animations for users who request it.
  • Never rely on animation as the sole way to communicate a state change.
  • Avoid content that flashes more than 3 times per second (WCAG 2.3.1 — Seizure Prevention).
  • See Motion for duration and easing token implementation.

Touch Targets

Standard Minimum Size Note
Recommended (WCAG 2.5.5) 44×44px Strongly recommended — avoids mis-taps on mobile
Minimum (WCAG 2.5.8 AA) 24×24px Only acceptable with adequate spacing between targets

Provide adequate spacing between adjacent touch targets. Icon buttons in toolbars or action bars should use 44×44px or add transparent padding to reach that size.

Language Attributes

  • Set the lang attribute on <html> for the primary page language: lang="en" or lang="ja".
  • For mixed-language content, apply lang on the inline element: <span lang="ja">. This helps screen readers switch pronunciation engines correctly.
  • See Typography for font family rules per language.

Do / Don't

Do
  • Target WCAG 2.1 AA as the minimum
  • Verify color contrast before finalizing palette
  • Use semantic HTML elements first
  • Provide visible focus indicators
  • Trap focus in modals and return on close
  • Label all form inputs and icon-only buttons
  • Apply prefers-reduced-motion
Don't
  • Remove focus outlines without a custom replacement
  • Use color as the only state indicator
  • Add ARIA roles when a native HTML element exists
  • Use tabindex > 0 — it breaks natural tab order
  • Add decorative icons without aria-hidden="true"
  • Make interactive targets smaller than 24×24px
