Global — Overview

Shared foundations, components, data conventions, and layout patterns used across all Colorkrew products. Everything here is the single source of truth — product specs reference global, they do not duplicate it.

Design system foundation

Global is built on MUI v7 (Material UI). All component documentation uses MUI's token system — palette.*, typography.*, spacing(n), shape.borderRadius, shadows[n] — rather than hardcoded values. Product-level overrides are documented in each product's _theme.md and _components.md, not here.

The mui-design Claude Code skill encodes all Global section rules for AI-assisted documentation. Load it when writing or reviewing any global page.

How products use Global

  • Foundations — products inherit global foundations and document overrides in _theme.md. If a product uses a different font or brand color, it lives in _theme.md, not as a fork of this page.
  • Components — product specs reference the global component page as the source. If a product customizes a component (e.g. a custom button variant), the customization is in _components.md and references this page for the base.
  • Guidelines & Glossary — global/_guidelines.md is the baseline. Each product's config/_guidelines.md extends or overrides it. If a product-level rule contradicts a global rule, it is intentional and must be explicitly noted.
  • Promotion — when a pattern appears inline in 3+ product specs, run /promote-to-global to move it here. The skill handles the move and updates all affected product references.
