Global — Changelog

Version history for shared Foundations and Components. Tracks design token updates, new components, and cross-product changes.

v0.6.0 2026-04-30 Minor
Foundations
Added
Elevation — shadow scale and z-index layers
New foundation page covering shadows[0]–shadows[24] and all seven named MUI z-index layers (mobileStepper through tooltip). Includes usage rules for elevation-based depth and stacking order.
Added
Breakpoints — responsive layout scale
New foundation page covering MUI's five breakpoints (xs 0px → xl 1536px), mobile-first documentation rules, and common layout patterns per breakpoint (drawer behaviour, grid columns, content offset).
Removed
Notification Patterns — retired
Replaced by the Toast (Colorkrew composite) and Snackbar / Alert component pages, which cover all notification patterns with MUI-accurate documentation.
Components — Added
Added
Icon Button, Rating — new Input components
Icon Button: compact icon-only action with required Tooltip pairing rule. Rating: star-based evaluation input with precision, readOnly, and accessibility label options.
Added
Typography — new Data Display component page
Documents the Typography component (variant + component prop decoupling, color tokens, use cases). Companion to the Foundations → Typography token page.
Added
Skeleton — new Feedback component
Content placeholder during loading. Covers text, rectangular, rounded, and circular variants; pulse/wave/false animation modes; sizing rules and when to prefer over LinearProgress.
Added
Paper — new Surfaces component
Base elevated surface underlying most MUI containers. Documents elevation 0–24, outlined variant, and the requirement to wrap Table in TableContainer component={Paper}.
Added
Bottom Navigation, Speed Dial — new Navigation components
Bottom Navigation: mobile primary nav (3–5 items, replaces drawer at xs–sm). Speed Dial: expandable FAB for 2–6 related actions. Both include product variation notes and do/don't guidance.
Added
Box, Stack, Grid — new Utilities layout components
Box: generic sx-prop container. Stack: one-dimensional flex layout with theme-spaced gaps. Grid: 12-column responsive grid with per-breakpoint column spans. All three include when-to-use-vs-each-other guidance.
Components — Reorganised
Changed
Nav structure fully aligned with MUI v7 catalog
App Bar moved from Surfaces → Navigation. Date Picker, Date Calendar, Time Picker moved from Utilities → Inputs. Modal added to Utilities nav. Nav Shell, Toast, Data Viz Primitives, and Form Validation grouped under a new Colorkrew sub-section.
Components — Removed
Removed
Backdrop, Input, Menubar, Number Field, Textarea Autoresize — retired
Backdrop and Input are internal MUI primitives not used directly in design. Menubar and Textarea Autoresize are MUI Base UI components not in the MUI v7 standard catalog. Number Field is MUI Base UI only. All replaced by or subsumed into their parent components.

v0.4.0 2026-04-30 Minor
Skills
Changed
figma-use — updated cross-reference and index filename
Cross-reference updated from figma-generate-design to figma-generate-library to match the installed skill name. Fixed plugin-api-standalone.index.md filename (was plugin-api-standalone_index.md).
Added
figma-generate-library — design system builder skill
New skill for building and updating professional-grade design systems in Figma. Orchestrates 20–100+ use_figma calls across 4 phases (Discovery → Foundations → File Structure → Components). Includes 7 reference docs and 9 reusable helper scripts. Works alongside figma-use.
Added
figma-create-design-system-rules — project rules generator
New skill that generates project-specific design system rules for Claude Code, Codex CLI, or Cursor. Analyzes the codebase and produces rules that encode component organization, styling conventions, design token usage, and Figma-to-code workflow for the active project.
Added
mui-design — MUI design system foundations skill
New skill synthesized from the MUI v7 and MUI Base UI packages. Covers the full component catalog (39 components across 6 categories), theme token system (palette, typography, spacing, shape, breakpoints, z-index, transitions, shadows), component design rules, Figma-to-MUI token mapping, accessibility requirements, and spec-writing conventions. Serves as the design-layer foundation for Global section documentation.
Changed
audit-figma, document-feature-spec, cross-product-check — skill references added
All three skills now explicitly reference loading figma-use before Figma MCP calls, and mui-design for component and token validation. Reduces repeated prompting and ensures consistent Figma protocol adherence.
v0.3.0 2026-04-29 Minor
Foundations
Changed
All 6 foundation pages fully authored
Color System, Typography, Spacing, Motion, Iconography, and Accessibility pages replaced with complete, content-rich guides. Each page includes MUI token references, usage rules, visual examples, and do/don't guidance. Companion .md files optimized for AI reference.
Data
Changed

Global Styling
Changed

All pages now use a plain muted description paragraph instead of the blue blockquote alert box. The divider between title and description was also removed. Code snippets in descriptions now render with the blue monospace style consistent with body content.
Components
Changed
Component library completely rewritten — 39 MUI-based components
Replaced 9 legacy placeholder components (Button, Input, Modal, Table, Nav Shell, Toast, Pagination, Data Viz, Form Validation) with 39 fully documented MUI components organized into 6 categories: Inputs (12), Data Display (7), Feedback (5), Surfaces (3), Navigation (7), Utilities (5). Each component includes variants, sizes, states, colors, key options, and common use cases. Designer-focused — no implementation code.
