Box

Generic layout container with full MUI sx prop support. The most flexible layout primitive — use Box when you need an element that responds to theme tokens but doesn't require a semantic MUI component.

Utilities
MUI composition: Box renders as div by default. Change the rendered element with the component prop (span, section, main, etc.). Supports the full sx prop for theme-aware styling.

Key Props

PropPurpose
componentOverride rendered HTML element (default: div)
sxTheme-aware inline styles (padding, margin, color, etc.)
displayShorthand for display: flex, grid, inline, etc.

When to Use Box

  • When you need a layout wrapper with sx prop access
  • When no semantic MUI component fits (not a Card, not a Paper)
  • As a flex or grid container when Stack/Grid is insufficient

When NOT to Use Box

  • For flex one-direction layouts → use Stack
  • For 12-column responsive grids → use Grid
  • For elevated surfaces → use Paper
Customization: Visual styling and theme overrides belong in the product's _theme.md. Component-level variants and usage rules specific to a product belong in _components.md.
