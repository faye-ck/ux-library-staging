Skeleton

Content placeholder shown during loading. Prefer Skeleton over spinner for list views, cards, and content-heavy areas — it reduces perceived load time by hinting at the shape of incoming content.

Feedback
MUI composition: Skeleton is a standalone component. Use inside a conditional — render Skeleton while data is loading, replace with the actual content when loaded.

Quick Reference

Variants

text (default, height matches font-size)
rectangular (fixed height, use for images and cards)
rounded (rectangular with border-radius)
circular (use for avatars)

Animation

pulse (default — fade in/out)
wave (shimmer effect)
false (no animation — for reduced motion)

Key Options

  • width / height — explicit sizing (required for rectangular and circular)
  • variant — shape of the placeholder
  • animation — pulse / wave / false
  • Match skeleton dimensions to the real content it replaces

Common Use Cases

  • List items loading (match height of each row)
  • Card image placeholders (rectangular)
  • Avatar placeholders (circular)
  • Text line placeholders (text variant, multiple lines)

What NOT to use for: action buttons or interactive elements that aren't data-dependent; use LinearProgress for indeterminate page-level operations instead.

Customization: Visual styling and theme overrides belong in the product's _theme.md. Component-level variants and usage rules specific to a product belong in _components.md.
