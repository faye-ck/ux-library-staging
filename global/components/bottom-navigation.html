Bottom Navigation

Mobile primary navigation bar fixed to the bottom of the screen. Replaces the side drawer on small screens when there are 3–5 top-level sections.

Navigation
MUI composition: BottomNavigation + BottomNavigationAction. Each action requires a label and icon. The bar sits at z-index: appBar (1100) and is typically placed inside the Nav Shell on mobile breakpoints.

Quick Reference

Item Count

Minimum: 3 items
Maximum: 5 items
2 items → use Tabs instead

States per Action

Inactive (icon + label)
Active / selected (color: primary.main)
Disabled

Key Options

PropValues / Notes
showLabelsAlways show labels regardless of selection state (recommended)
value / onChangeControlled selection state
Fixed bottom positionUse position: fixed; bottom: 0 or MUI's BottomNavigation with Paper elevation

Common Use Cases

  • Mobile Web primary navigation (replaces drawer below sm breakpoint)
  • Mobile App primary sections (Home, Search, Notifications, Profile)

Product Variations

  • Mobile Web: 4 primary sections — exact labels defined in product _layout.md
  • Not used on desktop — drawer takes over at md breakpoint
Customization: Visual styling and theme overrides belong in the product's _theme.md. Component-level variants and usage rules specific to a product belong in _components.md.
