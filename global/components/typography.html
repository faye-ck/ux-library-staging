Typography

The Typography component applies MUI's type scale to any text element. Maps directly to the typography tokens defined in Foundations — use the component to render text consistently across the product.

Data Display
MUI composition: Typography renders a semantic HTML element (h1–h6, p, span) chosen via the component prop, styled to a type scale variant via the variant prop. The visual style and the semantic tag are independent.

Quick Reference

Variants (map to HTML)

h1–h6 → heading tags
subtitle1, subtitle2 → h6 / span
body1, body2 → p
caption → span
overline → span (uppercase)
button → span (button label)

Colors

inherit (default)
primary (palette.text.primary)
secondary (palette.text.secondary)
disabled (palette.text.disabled)
error (palette.error.main)

Key Options

  • variant — the type scale token
  • component — the rendered HTML element (defaults vary by variant)
  • noWrap — single-line with overflow ellipsis
  • gutterBottom — adds bottom margin equal to 0.35em
  • paragraph — renders as <p> with paragraph spacing

Common Use Cases

  • Page and section headings (h1–h4)
  • Card titles (subtitle1), card descriptions (body2)
  • Helper text and captions (caption)
  • Category / status labels (overline)
  • All body copy (body1)

See Foundations → Typography for the full token scale and font specifications.

Customization: Visual styling and theme overrides belong in the product's _theme.md. Component-level variants and usage rules specific to a product belong in _components.md.
