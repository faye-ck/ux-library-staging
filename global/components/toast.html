Toast

Transient feedback notification — the Colorkrew pattern for Snackbar + Alert. Appears briefly, never blocks the user, and auto-dismisses after a fixed duration.

Feedback
MUI composition: Toast = Snackbar wrapping an Alert component. The Snackbar controls positioning and auto-dismiss timing; the Alert provides severity styling and the optional action button.

Key Options

  • Severity (success · info · warning · error) — sets icon and color
  • Action button — renders inside the Alert (e.g. Undo, Retry, View)
  • Close button — always show for error and warning; optional for success and info
  • Anchor position — bottom-right default; bottom-center on mobile (< sm breakpoint)
  • Auto-hide duration — set per severity (see Timing table above)
  • Stacking — queue multiple toasts; show one at a time, dismiss to reveal next

Bilingual rules (EN/JA)

  • Toast messages must be translated — no English-only toasts in Japanese UI
  • Japanese messages are typically longer; max character count: 60 JP / 80 EN per line
  • Action button labels: keep to 4 characters or fewer in Japanese (e.g. 元に戻す → 取消)

Common Use Cases

  • Save confirmation — "Changes saved" (success, 4 s)
  • Delete confirmation with undo — "Deleted" + Undo action (6 s)
  • Copy to clipboard — "Copied to clipboard" (success, 2–3 s)
  • Network error — "Failed to save. Please try again." (error, 8 s)
  • Permission denied — "You don't have access." (warning, 6 s)
  • Background task complete — "Export ready. Download" (info + action)

What not to use Toast for

  • Form validation errors — use inline Alert or TextField helper text instead
  • Confirmations requiring a decision — use Dialog instead
  • Persistent system status — use an inline Alert at the top of the page
  • Critical destructive results — if the action cannot be undone, use a Dialog to confirm first
Customization: Anchor position defaults and auto-hide durations belong in the product's _components.md. Severity color overrides belong in _theme.md.
