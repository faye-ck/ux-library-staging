Nav Shell

The application navigation frame shared across all Colorkrew portals — composed of AppBar + Drawer. Defines where global nav, product nav, user actions, and content live at every breakpoint.

Navigation
MUI composition: Nav Shell = AppBar + Toolbar (top bar) + Drawer (side panel). The AppBar position is fixed; body content has top padding equal to the toolbar height. Drawer variant depends on breakpoint.

Key Options

  • Drawer width — product-defined in _theme.md (typically 240–280px expanded, 56–72px mini)
  • Drawer collapse — persistent drawer can collapse to icon-rail; state persisted in localStorage
  • Active nav item — use selected prop on ListItemButton; match on current route
  • Nested nav items — one level of collapsible sub-items allowed; second level must be a separate page nav
  • User avatar menu — Menu anchored to avatar; contains Profile, Settings, Sign out
  • Top bar elevation — AppBar elevation={0} with border-bottom is the Colorkrew default (flat style)
  • Mobile swipe — Drawer onClose handles backdrop tap and swipe-to-close on mobile

Product variations

  • Biz portal — persistent left drawer with product switcher at top; mini-rail default on md
  • Mobile Web — top bar only; bottom navigation replaces drawer for primary sections
  • Admin Portal — full persistent drawer; secondary admin sections in a nested list
  • CKID / Intra — same shell structure; brand color applied to AppBar via _theme.md
Customization: Drawer width, AppBar background color, and mini-rail behavior are product-specific and belong in _theme.md. Nav item structure (links, labels, nesting) belongs in _layout.md for each product. This page documents the shared shell pattern only.
