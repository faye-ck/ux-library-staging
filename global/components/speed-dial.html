Speed Dial

Expandable FAB that reveals a set of related actions. Use when a single FAB isn't sufficient and actions share a common context — typically 2–6 related actions.

Navigation
MUI composition: SpeedDial + SpeedDialAction. Built on Fab. Opens on hover (desktop) or click/tap (mobile). Each action requires a name (tooltip) and icon. Z-index: speedDial (1050) — below AppBar.

Quick Reference

up (default)
down
left
right

Closed (shows main FAB)

Action hover

Key Options

PropValues / Notes
directionWhere actions expand: up / down / left / right
open / onOpen / onCloseControlled state
hiddenHides the entire SpeedDial (use for scroll-based show/hide)
SpeedDialAction: tooltipTitleRequired. Also accepts icon and onClick

Common Use Cases

  • Create actions (new task, new event, new message)
  • Context-specific quick actions
  • Mobile-primary workflows where screen space is limited

What NOT to Use For

  • Primary page-level actions that should always be visible — use a regular button instead
  • Actions that apply destructive changes without confirmation
