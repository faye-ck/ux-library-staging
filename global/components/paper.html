Paper

Flat elevated surface that underlies most MUI containers. The base layer for cards, dialogs, menus, and any content region that needs visual separation from the page background.

Surfaces
MUI composition: Paper renders a div with palette.background.paper fill and a shadow from shadows[elevation]. Most higher-level components (Card, Dialog, Menu, Drawer) are built on Paper.

Quick Reference

Elevation Levels

0 — flat, no shadow (same as background.paper)
1 — subtle lift (default)
3 — menus and popovers
8 — dialogs and modals
24 — maximum depth

Variants

elevation (default — uses shadows scale)
outlined (border instead of shadow, no elevation)

Key Options

PropValues / Notes
elevation0–24 (default: 1)
variantelevation / outlined
squareDisables border-radius (default uses theme.shape.borderRadius)
componentOverride the rendered HTML element

Common Use Cases

  • Wrapping Table (always use TableContainer component={Paper})
  • Content panels and sections
  • Bottom sheets on mobile
  • Form containers

What NOT to Do

Don't apply Paper to every div — use it only where visual separation from the background is needed.

Customization: Visual styling and theme overrides belong in the product's _theme.md. Component-level variants and usage rules specific to a product belong in _components.md.
