Figma Tips

How Claude reads your Figma files — and how to get the best results from every skill.

How Claude reads Figma

Every skill that reads Figma follows the same four-step protocol, powered by the figma-use skill. Understanding this helps you structure your files so skills extract accurate, complete information the first time.

The figma-use skill is a reference library installed in .claude/commands/ that provides Claude with the full Figma Plugin API reference, common patterns, gotchas, and design system guides. It is not invoked directly — it shapes how Claude reads and interprets Figma data in every skill that uses a Figma URL.

1
Inspect with get_metadata
Always called first. Reads the full node hierarchy — frame names, page count, structure. Claude uses this to understand what's in the file before fetching any content. Never skip this step or call get_design_context blind on a large frame.
2
Extract with get_design_context
Called on individual screens — not the root frame. For multi-screen files, Claude calls separately per section to avoid truncation. This gives Claude the UI copy, layer names, component types, and annotations per screen.
3
Verify with get_screenshot
Always taken after extraction. Used to visually confirm layout, catch states that are hard to infer from metadata (hover, error, disabled, empty), and spot designer annotations. Screenshots are especially important for delta work — comparing platforms side by side.
4
Work incrementally
If get_design_context is truncated, Claude notes the missed screens, calls again on those specific nodes, and lists any still-unread screens in the Open Items section of the output.
Figma URL format

The URL you provide determines how much context Claude pulls. More context isn't always better — large frames can exceed token limits and reduce accuracy.

✓ Recommended — Specific frame URL
https://www.figma.com/file/XXXX/Name?node-id=123%3A456
Targets one frame. Fewer tokens, more accurate extraction. Use this whenever possible. Copy from Figma by right-clicking a frame → "Copy link to selection".
△ Acceptable — Prototype URL
https://www.figma.com/proto/XXXX/...
Works for visual context but metadata is limited. Claude can see the screens but may have less access to layer names and component data.
✕ Avoid — Full file URL
https://www.figma.com/file/XXXX/Name
Pulls everything in the file. Can exceed token limits on large files, causing truncation. Only use for very small, single-screen files.
How to structure your frames

Claude reads layer names and frame names to understand context. Well-named frames produce better specs with fewer open items.

  • Name your frames descriptively. Use names like "Empty State", "Error — Network", "Admin View" rather than "Frame 123". Claude uses these names to identify screen purpose.
  • Group screens by platform or state. If your file covers multiple platforms (User Portal, Mobile Web), put each in a named top-level frame. Skills like /add-platform-delta need to target them individually.
  • Use component names from the glossary. If a component is called "Toast" in the library, name the layer "Toast" in Figma. Mismatched names appear as undocumented components in Open Items.
  • Annotate intentional deviations. If a platform omits a feature intentionally, add a text layer or sticky note explaining why. Claude will include it in the spec and reduce false-positive Open Items.
  • Keep one feature per frame link. Passing one Figma URL per feature (or per platform) gives the cleanest extractions. Avoid linking to pages with many unrelated features mixed together.
  • Show all states you want documented. Empty, loading, error, and success states must be visible as separate frames or sections. If a state isn't in the Figma file, Claude can't document it and will flag it as missing.
Common gotchas
Truncated extraction on large frames
If a frame has 20+ screens, get_design_context may be cut off. Claude will note this in Open Items and re-call on missed nodes — but you'll get faster, more complete results by linking to individual sections rather than one giant page.
Hidden layers aren't extracted
Layers hidden in Figma are not included in get_design_context. If you have hidden states (e.g. a disabled button variant), make them visible before running a skill or show them in a separate frame.
Hex values instead of token names
If a color in Figma is applied as a raw hex (e.g. #2563EB) instead of a library variable, Claude will flag it as an inconsistency in Open Items. Use Figma library color variables so Claude can map them to token names in _theme.md.
Unapproved copy terms
Copy in Figma is compared against _glossary.md during spec generation. Terms not in the glossary (e.g. "Cancel" vs the approved "Dismiss") are flagged as inconsistencies. Check the product glossary before writing UI copy in Figma.
Platform-specific frames on the wrong page
For /add-platform-delta, you need separate Figma URLs for baseline and delta platforms. If both are on the same page with no clear separation, Claude may mix up which screens belong to which platform.
