Template Layout — Feature

Step 1 of 3 in the template creation wizard. Admin builds a workflow form by dragging layout sections and form elements onto a canvas, then configuring each element's settings.

Platform: User Portal  ·  Role: Admin (template creator) inferred  ·  Wizard step: 1 / 3 (Layout)

Layout

Total canvas width: 1600px. Three fixed columns split by the stepper at top.

Left panel 330px Stepper · Form picker · Layout tiles · Element palette
Main canvas 940px (scrollable) Subject fields · Sections · Elements max-width 960px · 24px padding
Right panel 330px Element settings (empty by default)

The top bar spans the right 1270px: breadcrumbs on the left, action footer (status + buttons) on the right. The left panel (330px) is fixed and sits outside the top bar.

Core Flows
  • 1Start from scratch (empty canvas) — Admin lands on Layout step. Stepper shows ① Layout (active, blue underline) | ② Approvals | ③ Preview. Canvas displays: "Drag and drop a layout section from the left panel to get started." Right panel shows empty state. Action bar: Cancel | Save as Draft | Publish (disabled).
  • 2Select form type — Admin picks a form from the "Form" dropdown in the left panel (e.g. "Custom form"). inferred — gates available element types.
  • 3Add subject fields — Admin clicks "Add subject fields" (Select with Chips) at the top of the canvas. Hint: "Select up to 3 fields, with at least 1 required field." confirm Is 3 a hard limit?
  • 4Add a Section — Admin drags "Section" tile from left panel to canvas. Section appears with: "Enter section title*" input · "Enter description (optional)" · drop zone "Drag and drop to add element here." Header controls: ⠿ drag handle | 🔔 pin | 🗑 delete | ∧ collapse.
  • 5Add a Multi-entry Section — Same as above but adds a "Max. entries: 10 ↓" control in the section header. confirm Is 10 a fixed default or user-configurable?
  • 6Add a form element — Admin drags an element tile (e.g. Short Text) from the palette to a section drop zone. Element appears in a 2-col grid inside the section. Element is auto-selected; right panel opens with that element's settings.
  • 7Configure element settings — Right panel shows the selected element's config. For Dropdown: title "Dropdown" + ✕ close · Required field toggle · Label title* text field · Allow multiple selections checkbox · Options list (min 2, each with ⠿ | text field | + | 🗑) · Add option button · "All fields are required" error · Remove element (red link).
  • 8Reorder via drag-and-drop — 5 snapping zones: (a) in between rows, (b) left of a row, (c) right of a row, (d) left side of an element (creates 2-col split), (e) right side of an element (creates 2-col split). Each zone has an animated snap indicator.
  • 9Collapse / expand section — Admin clicks ∧ icon. Section collapses: title stays visible, description truncates to 1 sentence with ellipsis, elements hidden. Click again to expand.
  • 10Delete section — Admin clicks 🗑. Dialog: "Delete section?" / "All elements inside will be permanently deleted. This action cannot be undone." → [No, keep section] [Yes, delete section].
  • 11Save as Draft — Admin clicks "Save as Draft." Action bar updates: status badge "Draft ·" appears, buttons become Cancel | Save Changes | Publish.
  • 12Edit saved draft with unsaved changes — Admin edits after saving. Action bar: "Unsaved changes · Draft ·" | Cancel | Save Changes (primary, blue) | Publish.
  • 13Exit with unsaved changes — Admin clicks Cancel or navigates away. Dialog: "Exit without saving?" / "You have unsaved changes. Leaving now will discard them permanently." → [No, keep editing] [Yes, discard changes].
  • 14Validate on Publish — Admin clicks Publish with incomplete fields. Required field labels highlight in red: "Missing input*". Empty section drop zone: red dashed border + "At least 1 element is required." Options fields: "All fields are required" in settings panel.
