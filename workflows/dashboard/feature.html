Dashboard — Feature

Home page for Colorkrew Workflows. Entry point for all submitters and approvers to view, filter, and act on workflows.

Core Flows

Flow 1 — Submitter reviews own submissions

1Lands on Dashboard (Home active in left nav)
2Greeting: "Welcome back [First Name]!" + "You have [N] actionable items and [N] pending submissions"
3Default tab: "To approve" shown first
4User reads table or card list; can sort by column in table view
5User clicks row or card → Details modal opens (overlay)

Flow 2 — Approver acts on pending approval

1Lands on Dashboard; "To approve [N]" tab shows count
2Clicks a workflow row/card → Details modal opens
3Reviews form sections and right-panel Approval timeline
4Clicks Approve (green) or Reject (red dropdown) in Action Footer
5Item transitions out of "To approve" → appears in "Recently actioned"

Flow 3 — User filters by tab

1User clicks any tab: To approve / Recently actioned / Need revision / Pending / On hold / Approved / Draft
2List refreshes; active tab shows underline indicator and updated count badge

Flow 4 — Toggle table ↔ card view

1User clicks view-toggle icons (top-right of content area)
2Table view: MUI Data table, sortable columns (no standalone Sort dropdown), pagination
3Card view: Grid of cards, infinite scroll (20–30 initial, loads next on scroll-to-bottom)
4On XS (≤519px): only card view available, toggle hidden

Flow 5 — Open workflow details

1User clicks a table row or card
2Details modal opens as overlay on Dashboard
3Breadcrumb: "Home / [Tab name] / [Ref. No.]" with edit icon; expand and close buttons
4Left section: workflow form (collapsible sections) + sticky Action Footer
5Right section (330px): Approval / History / Comments tabs with timeline
6Close (×) → returns to Dashboard; scroll position preserved INFERRED

Flow 6 — Manage right panel mentions

1Right panel shows "Mentioned [N]" with count badge
2Unread items shown bold with red dot
3Hover item → "clear" icon overlaps right edge → click removes item (not deleted; revisit via Inbox)
4"Mark all as read" link → marks all read. "Clear list" link → clears all.
5Click item body → navigates to referenced workflow

Flow 7 — Admin view

1Admin sees CTA "+ Create / Manage" instead of "+ Create"
2Extra "To update [N]" tab when proxy user assignment is pending

Flow 8 — Mobile web (XS, ≤519px)

1No sidebar; hamburger opens nav drawer; "Left sidebar" icon → "burger menu" icon
2Grid view only; no view toggle; floating "+ Create" button bottom-right
3Tabs become sticky (float at top as user scrolls); 16px horizontal margin

Empty State component (254px height) shown when selected tab has no workflows. Title: "You don't have any workflows yet". Subtitle: "You'll see them listed once there's something to submit or review". Appears inside a container frame at 960px width. Focus management (modal)

  • Open: move focus to first interactive element (close button or workflow title)
  • Close: return focus to the triggering row or card
  • While open: trap focus inside modal
  • Escape must close the modal

  • Icon-only buttons (modal close, link copy, expand, view toggle, mention clear): must have aria-label
