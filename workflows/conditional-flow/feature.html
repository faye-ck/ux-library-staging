Conditional Flow — Feature

User Portal baseline (Web desktop) · IF/THEN routing rules that determine which approval flow activates based on form field values.

Config files are stubs. All color token names, copy terms, and guideline references are marked CONFIRM throughout this spec. Populate workflows/config/_theme.md, _glossary.md, and _guidelines.md to resolve them.

Overview

Conditional Flow is a template-level configuration nested inside the Approval Flow Setup page (step 2 of 3 in template creation). An admin defines one or more IF/THEN condition flows. When a submitter fills in a workflow form, the system evaluates conditions top-to-bottom — the first matching flow's approval chain activates. If no condition matches, the default approval flow applies.

All configuration happens in the "Condition Setup" right panel. The approval flow chart in the canvas updates reactively as condition flows are added, reordered, or removed.

Core Flows

Flow 1Access Condition Setup
  1. Admin reaches step 2 (Approval Flow Setup) in template creation wizard.
  2. Admin activates the Conditional Flow configuration. CONFIRM exact trigger — button or toggle in main canvas.
  3. "Condition Setup" right panel opens.
  4. If no conditions exist: empty state — "No condition set" (italic, muted text). CONFIRM
  5. "Add condition flow" outlined button visible at panel bottom.
Flow 2Add a Condition Flow (first)
  1. Admin clicks "Add condition flow" button.
  2. "Condition flow 1" section appears with: section header (grip · label · duplicate · trash · chevron) · IF field selector · one empty group box · "Add condition" link · "Add group" link · THEN approval flow selector with edit icon.
  3. Admin selects a field from the IF field selector.
  4. A condition row (operator dropdown + value input) appears inside the group.
Flow 3Add More Conditions (AND/OR within a group)
  1. Admin clicks "Add condition" inside a group.
  2. New condition row appended. AND/OR toggle appears between conditions.
  3. Toggle default: AND. INFERRED
  4. All conditions in the group share the same AND/OR logic — mixed logic (e.g. A+B/C without grouping) is intentionally prohibited to avoid ambiguity about operator precedence.
Flow 4Add a Group
  1. Admin clicks "Add group" link below the existing group box.
  2. New group box appended. AND/OR toggle appears between groups.
  3. All groups in the condition flow share the same top-level AND/OR toggle (same constraint as within-group logic).
Flow 5Change AND/OR Toggle
  1. Admin clicks the inactive option on an AND/OR toggle.
  2. Toggle updates; all rows/groups in scope switch to the new logic.
  3. Any second AND/OR toggle in the same scope automatically follows the first.
Flow 6Add Additional Condition Flows
  1. Admin clicks "Add condition flow" below all existing flows.
  2. New "Condition flow N" section appended. Evaluation order: top-to-bottom, first-match wins.
  3. Approval flow chart re-renders to reflect the new flow.
Flow 7Duplicate a Condition Flow
  1. Admin clicks the duplicate icon in a section header.
  2. Copy created below original with same IF conditions and THEN approval flow.
  3. Section label auto-increments to "Condition flow N+1". INFERRED
Flow 8Reorder Condition Flows (Drag and Drop)
  1. Admin grabs grip icon in a section header and drags to new position.
  2. On drop, evaluation order updates. Flow chart re-renders to reflect new order.
Flow 9Remove a Condition or Group
  1. Admin clicks trash icon on a condition row. Row removed immediately.
  2. CONFIRM Does group box also have a remove control? Does an empty group auto-remove or persist?
Flow 10Remove a Condition Flow
  1. Admin clicks trash icon in a condition flow section header.
  2. Entire IF/THEN block removed. Remaining flows renumber. INFERRED
Flow 11Remove All Conditions
  1. Admin triggers "Remove all". CONFIRM exact location (overflow menu or panel header control).
  2. Basic Dialog confirmation appears. Copy: MISSING
  3. Admin confirms → all flows cleared, panel returns to empty state.
Flow 12Expand / Collapse Condition Flow Sections
  1. Admin clicks chevron icon in section header → section collapses to header-only.
  2. Admin clicks collapsed header → section expands.
  3. Multiple sections can be independently expanded/collapsed.
Flow 13Conditional Approval Flows Reflected in Left Panel
  1. With only the default approval flow: no "CONDITION APPROVAL FLOW" section visible in the Approval Flow Setup left panel.
  2. When at least one conditional flow has a THEN approval flow assigned: "CONDITION APPROVAL FLOW" section appears.
  3. Each conditional AF shown as a collapsible section: section header · description · Self-approval toggle · Steps with User list.
  4. Multiple conditional AFs can be independently expanded/collapsed.
  5. Scroll indicator appears on right side when content overflows.
Flow 14Maximum Limit Reached
  1. Admin reaches the maximum number of conditions or condition flows.
  2. "Add condition" / "Add condition flow" controls become disabled or hidden. CONFIRM exact behavior and limits.

Edge Cases
  • Single condition in a group: AND/OR toggle not shown. INFERRED — toggle only appears at 2+ conditions or groups.
  • Reused approval flow nodes in chart: positioned lowest so all conditional paths can reach them. Forward-arrow loops never occur (first-come-first-display rule).
  • All conditions removed: panel reverts to empty state.

Approval Flow Chart — Display Rules

Source: Figma node 3542-234440 (authoritative). The chart renders in the Approval Flow Setup main canvas and updates reactively as condition flows change.

  • Evaluation order: top-to-bottom, first-match wins.
  • Forward arrows (skipping to a later node) never occur — condition order determines activation.
  • When an approval flow is reused across multiple condition flows, the reused AF node is positioned lowest in the chart so it remains reachable from all paths referencing it.
  • Ordering rule: earliest reused AF node → lowest position; later reused AF nodes → progressively higher up to their original; last reused and non-reused AF nodes stay at their original positions.
