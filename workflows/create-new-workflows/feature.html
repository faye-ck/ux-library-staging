Create New Workflows — Feature

Entry point for submitters to initiate a new workflow. Two-phase flow: template selection dialog, then a full-screen form modal with dynamic approval flow panel, multi-section form, draft saving, and a review step before submission.

Layout
Phase 1 — Template selection dialog
  Basic Dialog: 600px centered · dimmed backdrop over Dashboard
  Category dropdown + Template dropdown + Cancel / Continue

Phase 2 — Details modal (1400px overlay)
  ├── Details header (56px) — breadcrumb + link / expand / close
  ├── Left section (1070px) — scrollable form + sticky Action Footer (60px)
  │   ├── Title (dynamic: Product : Environment : Feature)
  │   ├── Submitting as dropdown
  │   ├── Form sections (collapsible): Release Description · Security Check
  │   ├── Multi-entry section (template-dependent)
  │   └── Template info tab (read-only)
  └── Right section (330px) — Approval Flow panel (collapsible)

Responsive: MISSING — no responsive frames in section. Inferred: follows Dashboard modal patterns.
Core Flows
Flow 1 — Create new workflow (full happy path)
  1. User clicks "+ Create" button on Dashboard
  2. "Create a new workflow" dialog opens over Dashboard with dimmed backdrop
  3. User selects workflow category (optional filter) and template (required)
  4. User clicks "Continue" → dialog closes; Details modal opens (1400px)
  5. Modal breadcrumb: Home / Create New Workflow
  6. Title shows placeholder: Product Name* : Environment* : Features
  7. User selects "Submitting as" (org path + person; defaults to current user)
  8. User fills required fields (Product Name*, Environment*) → title updates dynamically
  9. User fills optional fields: Date and Time, Features, Remarks, Security Check
  10. Approval Flow panel populates from template configuration
  11. User clicks "Next" → review screen; right panel switches to Approval / History / Comments tabs
  12. User clicks "Submit Workflow" → workflow submitted Inferred
Flow 2 — Save as draft then resume
  1. User partially fills form and clicks "Save as Draft"
  2. "Draft" chip appears near modal title; footer button changes to "Save Changes"
  3. User exits modal → draft preserved in Dashboard Draft tab
  4. User re-opens draft; "Draft" badge visible; form fields editable
  5. User edits → "Unsaved changes" chip appears; "Save Changes" enabled
  6. User clicks "Save Changes" → changes persisted; "Unsaved changes" chip clears
Flow 3 — Conditional approver selection (single department)
  1. Template has an approval step requiring a choice from a named group
  2. Right panel shows step with dropdown: [Group Name] / Any of N approvers
  3. User opens dropdown → sees group members listed
  4. User selects specific approver (or keeps "Any of N approvers" default)
  5. Selection triggers loading: "Updating approval flow..." spinner; all footer buttons disabled
  6. Once resolved, right panel shows chosen approver; buttons re-enable
Flow 4 — Conditional approver selection (multi-department)
  1. Same as Flow 3 but approval step spans multiple departments
  2. Dropdown shows department sections (Department A, Department B) each with member list
  3. "Any of N approvers" option at top; pre-selected with checkmark (default)
  4. User selects a specific department member → approval flow updates
Flow 5 — Multi-entry section
  1. Template includes a multi-entry section (repeating row group)
  2. Section header shows "Max. N entries" badge; description text in teal color
  3. Each entry row: Item* (required text) + Price (¥ + number) + Description (textarea)
  4. User clicks "Add entry" → new empty row appended; button disabled at max
  5. User clicks delete icon on a row → row removed
States
Empty / Initial
  • Phase 1: both dropdowns empty; Continue button disabled Inferred
  • Details modal: title shows full placeholder text; "Next" disabled; "Save as Draft" enabled
  • Approval Flow: some steps may show empty avatar circle (unresolved approver)
Loading — Updating Approval Flow
  • Right panel replaced by spinner + "Updating approval flow..." text
  • All footer buttons (Cancel, Save as Draft / Save Changes, Next) disabled
  • Triggered by: conditional approver selection, certain field changes
Draft
  • "Draft" chip inline near modal title
  • Footer: "Save as Draft" → "Save Changes" (grayed out until edits made)
  • All form fields remain editable
Draft + Unsaved Changes
  • "Unsaved changes" chip + "Draft" chip both visible near title
  • "Save Changes" button enabled
  • "Next" enabled if required fields complete
Review (Details Edit — Next)
  • Step counter "Step 0/N" top-right of left section
  • Form fields rendered read-only (label + value pairs)
  • "Submitting as" shows full name + email (read-only)
  • Right panel tabs: Approval | History | Comments
  • Approval tab shows timeline: creation event + submission timestamp
  • Footer: "Back" | "Submit Workflow"
