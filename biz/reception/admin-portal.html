Security Card Management

Manages physical security cards — enable feature, create card inventory, pair cards with items, and monitor card status. Receptionist check-in/out flows: Receptionist Portal →

1 — Enable/Disable Feature

Location: Reception → Settings → Receptionist Settings → Security card reader

  • Toggle (default: Disabled). Enabling also shows "Also activate during check-out" toggle (default: Enabled).
  • When check-out enabled: return and check-out time recorded simultaneously. When disabled: recorded separately.
  • Save to apply. Once saved: Security Cards available in both portals; Guest Info gets "Status Remark" column.
⚠️ Disabling after data exists requires confirmation. Existing data preserved and restored if re-enabled.

2 — Create/Edit Security Cards

FieldTypeValidation
Item NameTextRequired · Non-unique · Max 50 chars
Item IDTextRequired · Non-unique · Max 50 chars
AccessDropdownOptional · From Access management config · Hidden if disabled

CSV bulk update: Columns: UUID · Item Name · Item ID · Access · Card Status (read-only). Preview before commit. Errors block submit; success shows created/updated counts.

Note: Assigned cards — Edit, Update Card ID, and Delete disabled. CSV edits for assigned cards ignored.

3 — Manage Security Cards

Tabs: Unpaired cards · Paired cards (with totals). CSV export from both tabs.

Unpaired Cards

Columns: UUID · Name · Item ID · Access · Pair card · Details (admin only). Single pairing: dialog → success auto-closes (3s). Bulk pairing: queue max 100; continuous scanning; errors: Try Again or Skip.

Paired Cards

Additional columns: Card Status · Assigned To · Host · Actions. Bulk action on checkboxes: Assigned → Return; Available → Update Card ID. Lost/Damaged: one-by-one only.
