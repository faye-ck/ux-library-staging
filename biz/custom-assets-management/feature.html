Custom Assets Management — Feature

User Portal baseline. Upload, rename, and remove 3D model assets (.GLB) and 2D thumbnail icons for the Office Map.

🟡 Draft product: biz · platform: user-portal · updated: 2026-05-01
Overview

Custom assets are decorative 3D model objects users can upload and manage for the Office Map. This page is the sole entry point for both creation and management — accessible via Office Map → Custom Assets tab in the User Portal. Assets are not functional like seats or spaces; they are purely visual map objects.

💡 Suggested: Feature lives within the Seat module context (left nav shows Seat as active). Confirm whether this is always the case or if the nav state varies by entry point.
Navigation path

Office Map (top nav) → Custom Assets tab

Tab strip (same level): Pages · Access Links · Custom Assets (active)

Core flows
Flow 1 — Upload a 3D model asset
  1. User opens Manage Custom Assets page (empty or populated list).
  2. User clicks Upload New (outlined button, top right of section header).
  3. File picker opens — accepts .GLB only; single file at a time.
  4. Upload begins; loading / skeleton state shown on new AssetCard.
  5. On success: new AssetCard appears in grid; Snackbar shows "Upload successful".
  6. Default asset name = filename minus .GLB (e.g., TV.GLB → "TV").
  7. If name already exists: suffix appended automatically (TV → TV 2 → TV 3…).
Flow 2 — Upload a 2D icon (thumbnail) for an existing asset
  1. User hovers over an existing AssetCard.
  2. Upload icon button appears overlaid on the thumbnail area.
  3. File picker opens — accepts .PNG or .JPG only.
  4. On success: thumbnail updates; 2D Icon file size shown in details row.
⚠️ Confirm needed: The guidance alert shows "maximum size of 500KB" for thumbnails; the error spec and error snackbar say 50KB. Which is the enforced limit?
Flow 3 — Rename an asset
  1. User hovers over AssetCard — pencil icon appears next to asset name.
  2. User clicks pencil → inline rename field activates.
  3. User edits and confirms name.
  4. If new name collides: suffix appended automatically (no error shown).
Flow 4 — Remove an asset
  1. User hovers over AssetCard → Remove Asset button appears.
  2. User clicks Remove Asset.
  3. Confirmation dialog opens (dark backdrop, modal).
  4. User confirms → asset removed; list refreshes.
🔴 Missing: Exact copy for the Remove Asset confirmation dialog (title, body, confirm CTA) not captured in Figma.
Flow 5 — Remove a 2D icon (thumbnail)
  1. User hovers over AssetCard thumbnail → remove icon appears.
  2. User confirms removal.
  3. 2D Icon field on details row reverts to "NA".
States
Empty state

Shown when no assets uploaded. Copy: "No custom assets here." The Upload New button remains in the section header — no secondary CTA inside the empty body.

Error states

All errors surface as Snackbar (bottom, error severity). Some validate before upload starts; others appear after.

Note: Dimension errors (icon too small/large) are handled by auto-fit to 32×32 — no error shown. Network/server errors collapse to "Upload failed".

Success / confirmation
  • Upload model → Snackbar: "Upload successful"
  • Upload icon → Snackbar: "Upload successful" (INFERRED)
  • Rename → Inline update, no snackbar
  • Remove asset → Card disappears; list count updates
Edge cases
  • Name collision on upload or rename: auto-suffix added (XXXX 2, XXXX 3…). No error shown.
  • 2D Icon not uploaded: details row shows "2D Icon: NA".
  • Total upload limit: 50MB (shown in guidance alert).

Design rules
AssetCard — hover-reveal controls
Context: custom-assets-management · 2026-04-30
  • Edit controls (pencil rename, Remove Asset button, thumbnail upload icon) are hidden by default and revealed only on card hover.
  • Thumbnail area shows upload overlay icon (semi-transparent dark circle, 50% opacity backdrop) on hover when no 2D icon is set.
  • Remove Asset button uses variant="outlined" color="error" at 32px height — smaller than the standard MUI 40px button.
Figma source: AssetCards component, "Controls on Hover" annotation.

Context: custom-assets-management (likely Office Map-wide) · 2026-04-30
