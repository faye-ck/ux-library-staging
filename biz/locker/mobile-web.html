1.1 Booked Locker — Start and Return

  • Booking exists → "Booked / [Locker name]" record shown in dashboard.
  • Booking starts automatically on the booked day at the configured start time.

Check-in before start time — reminder shown at check-in:

"You have a locker booking today in [locker group name] [locker name]. Please return before [return time]."
[Close] [Start Now]

Check-in after locker has started — reminder shown at check-in:

"You have a locker booking today in [locker group name] [locker name]. Please return before [return time]."
[Close]
  • Once started: "In use" row with locker name and return time. Tapping → details + Return button.
  • On return: "Returned / [Locker name]" record shown.
  • Reminder sent [#] min before return time (configured in locker group settings).

Check-out with in-use locker — return reminder shown:

"You have a locker booking today in [locker group name] [locker name]. Please return before [return time]."
[Close] [Return now]

1.2 Receiving a Transfer — Pick-up

  • Pick-up available → "Received / [Locker name]" record in dashboard.
  • Tapping → transfer details + Confirm pick-up button.
  • After confirming → "Picked-up / [Locker name]".

Overdue pick-up notification:

"You have an overdue delivery in [locker group name] [locker name]. Please confirm after pick-up."

2.2 Transfer Flow

  • Select size (hidden if only 1; default smallest; disabled if fully booked).
  • Select recipients: full company list (alphabetical); search by name/email; multiple selectable; avatar/name/role/remove; min 1 required.
  • Full form: selected size + recipient list (Edit button) + optional message + pin code (required; asterisk placeholder; "[#] characters are required").
  • Transfer button enabled when all required fields filled.
  • Confirmation: "Transfer Confirmed" + group/map/locker/coordinate + sender + date/time + message + recipients + pin code.

2.3 Book a Locker Flow

  • Select size (skipped if only 1; default smallest; disabled if fully booked).
  • Confirmation: "Booking Confirmed" + group/map/locker/coordinate + return time.
  • Dashboard shows "Booked / [Locker name]" and "In use" row with return time.
  • Warning if already active locker: "You already have an active locker. Please return it before booking a new one."

2.4 Scanning Dedicated Locker QR

  • Locker group info header.
  • Recipient prefilled (dedicated user/dept; no remove icon).
  • Optional message textarea.
  • Transfer button → confirmation screen: "Transfer Confirmed" + group/map/locker/coordinate + sender + date/time + message + recipients.
