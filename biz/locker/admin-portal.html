1.2 Layout Setup

  • Prefix (max 10 chars) · Suffix (max 10 chars) · Use leading zeros toggle (default off; scales with largest number) · All reflected immediately in grid preview.
  • Numbering Order: Left to right, left to right (default) / Left to right, right to left / Top to bottom, top to bottom / Top to bottom, bottom to top.
  • Locker Group Size: Rows × Columns · Default 4×4 · Range 1–25 · Reflected immediately in grid preview.
  • Font size scales with locker name: first 13 chars on one line; 14th char+ wraps to second line.

1.3 Grid Preview

  • Header: total cell count. Left: grid (columns A–Y, rows 1–25, each cell 96×64px). Right: selected cell settings (default first cell).
  • Cell size (max 3×3): merging starts top-left; cannot combine merged cells; errors block update.
  • Reducing Locker Group size affecting merged cells: error shown, grid not updated until resolved.
  • "Mark as non-locker" checkbox: dropdown — Empty / Touch panel / Rack / Utility / Other. Non-lockers excluded from locker count.
Layout cannot be modified after creation.

Locker Group Details Page

Figma →

2.1 Group Info

QR code, Group ID (UUID), Group Name, Area, Return Time (time + same/next day), Return reminder, Access Method, Created at. Actions: Edit Group · Print Group QR Code · Print All Locker Cards · Delete Group (confirmation required).

2.2 Locker Details Panel

Links: View locker list · Export to CSV · Update by CSV. Header shows total lockers + non-lockers. Status counts: Available · In use · Dedicated.

Same grid as creation. Per-locker details: coordinate, status, UUID, name, usage type (Public/Dedicated), pin by admin, Print Locker Card button. Non-locker: coordinate + label.

2.3 Edit Locker Details

  • Locker Name (text) · Usage Type (Public: 3 options / Dedicated: User / Dept; multiple via chips; NOT editable if "In use") · Locker Pin by Admin (alphanumeric; blank = no pin).
  • Changed cells highlighted yellow. Footer: "[#] changes made" · Cancel (confirm prompt) · Save Changes.
