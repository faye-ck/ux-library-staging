Locker Groups Section

Map list dropdown (all maps default). Grid of locker groups showing: name, map name, total lockers, In use / Dedicated / Available counts (only non-zero shown). Selecting → right panel with group details.

Office Map Integration

Search (Keywords)

Searchable: locker group name · locker name · dedicated user · dedicated department. Results appear under new Lockers tab in priority order:

  • Dedicated Lockers: User (avatar, name, locker, group, coordinate) → Make a Transfer. Department (dept icon, name, locker, group, coordinate) → Make a Transfer.
  • Public Lockers: By group name → full locker list (Show/Hide). By locker name → only matched locker listed.
  • Clicking a group name opens grid preview on map. Clicking a map name auto-loads that map.

Direct Click on Map

Click locker group 3D object → opens grid preview + right panel with group details and action buttons.

Locker Group Grid Preview

Header: group name. Counts: Total lockers left · [#] Available | [#] In use | [#] Dedicated right. Grid cells show locker name, QR icon (dedicated), "In use" label. Columns A–Y; rows 1–25.

Right Panel — Public Locker Group

Group name, map name, total lockers, QR code. Statuses: Available (with type breakdown) or "No lockers available" · In use · Dedicated. Only non-zero statuses shown.

Right Panel — Dedicated Locker

"Dedicated locker" label. QR code. Avatar/dept icon, locker name, user/dept name, role, coordinate.

Action Buttons

  • Make a Transfer — disabled if no Transfer-type lockers available.
  • Book Locker — disabled if no Bookings-type lockers available; hidden for dedicated lockers.

Make a Transfer Flow

  • Select size (hidden if only 1; default smallest; disabled if fully booked).
  • Recipients: search by name/email · multiple selectable · min 1 required.
  • Add message (optional) · Pin code (required; asterisk placeholder; "[#] characters are required").
  • Confirm enabled when all required fields filled.
  • Dedicated locker variant: no size/user search/pin; prefilled recipient (no remove); message only.
  • Confirmation: "Transfer Confirmed" + group/map/locker/coordinate + date/time + recipients + message + pin code. Locker highlighted in grid.

Book a Locker Flow

  • Select size (skipped if only 1; default smallest; disabled if fully booked).
  • Confirmation: "Booking Confirmed" + group/map/locker/coordinate + return time + status + booking date + started date.
  • Warning if already has active locker: "You already have an active locker. Please return it before booking a new one."

Locker Booking During Seat Booking

Enabled via admin setting. See Admin Portal settings →

Booking Flow

  • Checkbox "Add locker bookings for selected dates" (unchecked default) appears in seat booking form.
  • Shows existing locker bookings on selected dates: "You already have locker bookings on: [YYYY/MM/DD - Locker name]". If all dates booked → checkbox disabled.
  • If checked: Locker Group dropdown appears. Auto-assigned by priority: (1) same dept + same area + same map → (2) same dept + different area → (3) same area, same map → (4) different area, same map. If all same-map lockers fully booked: not auto-assigned.
  • Return time shown for selected group. Fully booked groups disabled in dropdown.
  • If updated dates all have bookings → checkbox auto-unchecked, dropdown hidden.

Fully Booked Handling

If selected group is fully booked for any dates: extra screen before seat confirmation (nothing created yet). Shows affected dates with empty locker group dropdowns. After resolving: seat + locker bookings created; seat confirmation shown.

My Bookings — Locker Tab

New "Locker" tab shown when there are in-use or future bookings.

List View

"In use" section first, then "[#] bookings" section. Each item: locker name, group name, map name, booking date, return time. Actions: In use → Return icon; Booking → Cancel icon.

Details Page

  • Group name, map name, locker name, coordinate, return time.
  • Status (In use / Booked) · Booking date · Return time · Started at (for in-use: auto or manual).
  • Return → "Return Confirmation" with Returned at date/time.
  • Cancel → confirmation dialog before cancelling.
